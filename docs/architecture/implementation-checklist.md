# Чек-Лист Реализации

Этот файл фиксирует рабочую последовательность реализации, чтобы не уходить в сторону. Его можно править по ходу проекта, но новые задачи должны либо попадать в один из пунктов ниже, либо явно быть вынесены в отдельный поток.

## Главный Фокус

Текущий фокус проекта:

`минимальный состав котельной -> инженерная модель -> проверка -> чертеж -> DXF -> evidence/report`

Минимальный состав:

- один котел;
- коллектор подачи;
- коллектор обратки;
- трубы;
- запорная арматура.

Пока этот поток не доведен, не уходим в двери, электрику, общую вентиляцию, красивый UI и другие соседние направления, если они не блокируют текущую цель.

## Рабочий Чек-Лист

- [x] Завести минимальный состав в общей доменной модели, без "рисунков сами по себе".
- [x] Сделать арматуру обычным оборудованием каталога и проекта.
- [x] Построить план, принципиальную схему и предпросмотр листа из одной модели.
- [x] Сделать DXF через `Project -> EngineeringDrawing -> CadDrawing -> Exporter`.
- [x] Добавить pilot evidence/status report.
- [x] Добавить список вопросов для закрытия перед инженерной проверкой.
- [x] Сформировать `паспорт проекта` для текущего сценария проверки.
- [x] Подтянуть реальные публичные источники по котлу, коллекторам, трубам и арматуре.
- [x] Закрыть автоматически всё, что честно закрывается из модели, расчета и источников.
- [ ] Подтвердить точные координаты патрубков котла по заводскому чертежу.
- [ ] Подтвердить точные координаты патрубков коллекторов по паспорту/чертежу.
- [ ] Довести проверку соединений до инженерного review report: что проверено, что не подтверждено, что требует расчета.

- [ ] Только после этого переходить к отдельному модулю нормативной проверки ГОСТ/СП по правилам и citations.

## Правило Принятия Задач

Перед началом новой задачи задаем один вопрос:

`Это приближает текущий минимальный состав к инженерной проверке и доказуемому отчету?`

Если ответ:

- `да` — задача идет в текущий поток;
- `нет` — задача откладывается;
- `частично` — сначала фиксируем, какой именно пункт чек-листа она закрывает.

## Что Считать Уходом В Сторону

Пока не закрыт текущий чек-лист, не уходим в:

- двери, окна, бытовую архитектуру помещения;
- электрику как отдельный большой раздел;
- общую газовую автоматику вне текущего минимального состава;
- новые типы оборудования, если не завершен базовый набор;
- сложную оркестрацию AI-агентов;
- косметический рефакторинг;
- улучшения интерфейса, не влияющие на инженерную модель и проверку.

## Как Пользоваться Этим Файлом

- После каждого заметного шага обновляем статусы пунктов.
- Если появляется новая обязательная подзадача, добавляем ее в список, а не держим "в голове".
- Если задача спорная, сначала привязываем ее к одному из существующих пунктов.
- Если пункт закрыт только частично, не отмечаем его выполненным до конца.
