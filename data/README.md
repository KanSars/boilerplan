# Evidence Data

Эта папка хранит структурированные данные для будущего evidence pipeline: документы-источники, требования, compiled rules и связи с элементами проекта.

Важно: данные в `data/evidence/typical-standalone-boiler-room` являются pilot dataset для типовой отдельно стоящей блочно-модульной котельной. Они помогают отладить форму данных и будущую оркестрацию AI-агентов, но не являются готовой нормативной базой и не подтверждают инженерное соответствие.

Правила для данных:

- Любой `Requirement` должен иметь хотя бы одну `DocumentCitation`.
- Любая citation должна ссылаться на существующий `SourceDocument`.
- Данные, полученные из публичных страниц, PDF или AI extraction, по умолчанию имеют статус `review_required` или `extracted`, но не `verified`.
- `verified` допустим только после отдельной процедуры проверки источника, версии документа, применимости и интерпретации.
- Для отдельно стоящего блока котельной всегда фиксируйте применимость через `ApplicabilityCondition`, а не свободным текстом.

Текущий pilot dataset намеренно ограничен: он описывает типовые сущности и несколько требований/оснований, чтобы приложение не превращало непроверенные нормы в "истину".
